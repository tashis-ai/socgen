## 📖 О проекте

**Генератор постов для соцсетей** — веб-приложение на Flask, которое превращает содержимое любой веб-страницы в готовый пост для публикации. Просто вставьте URL, выберите один из 8 стилей (от экспертного до ироничного), настройте уровень креативности — и получите структурированный текст за секунды.

Проект создан для быстрой адаптации контента под разные аудитории, автоматизации работы SMM и удобного тестирования промптов. Работает локально, поддерживает OpenAI-совместимые API, прокси, валидацию ссылок и полную обработку ошибок на клиенте и сервере. Исходный код открыт под лицензией MIT.


# Генератор постов для соцсетей

Веб-приложение на Python/Flask для создания постов в социальных сетях на основе содержимого любой веб-страницы. Парсит текст со страницы и генерирует пост в выбранном стиле через LLM API (OpenAI-совместимый).

## Возможности

- Ввод URL страницы с валидацией на клиенте и сервере
- 8 стилей поста: официальный, ироничный, молодежный, вдохновляющий, дружеский, провокационный, экспертный, эмоциональный
- Динамическая смена текста кнопки генерации при выборе стиля
- Регулировка креативности (temperature 0–2)
- Индикатор загрузки и блокировка формы во время запроса
- Копирование результата в буфер обмена
- Скачивание поста в TXT-файл с датой и временем в имени (`20260701_143045_post_Экспертный.txt`)
- Подтверждение сохранения с именем файла
- Модальные окна для всех ошибок с закрытием по кнопке, оверлею и Escape

## Установка и запуск

1. Клонируйте репозиторий и перейдите в каталог проекта:

   ```bash
   cd VCd03_postgen
   ```

2. Создайте виртуальное окружение (рекомендуется):

   ```bash
   python -m venv venv
   venv\Scripts\activate        # Windows
   # source venv/bin/activate   # Linux / macOS
   ```

3. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. Создайте файл `.env` на основе шаблона:

   ```bash
   copy .env.example .env       # Windows
   # cp .env.example .env     # Linux / macOS
   ```

5. Укажите в `.env` API-ключ и при необходимости прокси:

   ```env
   OPENAI_API_KEY=ваш-api-ключ

   # Опционально — прокси для запросов к сайтам и LLM API
   HTTP_PROXY=ваш-proxy
   HTTPS_PROXY=ваш-proxy
   ```

6. Запустите приложение:

   ```bash
   python app.py
   ```

7. Откройте в браузере: [http://localhost:5000](http://localhost:5000)

## API

### `POST /generate`

Генерирует пост на основе URL и стиля.

**Тело запроса (JSON):**

```json
{
  "url": "https://example.com/article",
  "style": "молодежный",
  "temperature": 0.8
}
```

**Успешный ответ (200):**

```json
{
  "post": "Текст сгенерированного поста...",
  "style": "молодежный"
}
```

**Ответ с ошибкой:**

```json
{
  "error": "Текст ошибки для отображения в модальном окне",
  "error_type": "http_403"
}
```

## Переменные окружения

| Переменная           | Обязательна | По умолчанию | Описание                          |
|----------------------|-------------|--------------|-----------------------------------|
| `OPENAI_API_KEY`     | Да*         | —            | API-ключ OpenAI (из `.env`)       |
| `LLM_API_KEY`        | Да*         | —            | Альтернатива `OPENAI_API_KEY`     |
| `LLM_API_ENDPOINT`   | Нет         | OpenAI Chat Completions | URL эндпоинта LLM API |
| `LLM_MODEL`          | Нет         | `gpt-3.5-turbo` | Модель для генерации          |
| `HTTP_PROXY`         | Нет         | —            | HTTP-прокси для всех запросов     |
| `HTTPS_PROXY`        | Нет         | —            | HTTPS-прокси (если не задан — используется `HTTP_PROXY`) |

\* Достаточно указать одну из переменных: `OPENAI_API_KEY` или `LLM_API_KEY`.

## Технические детали

- **Парсинг страницы:** requests + BeautifulSoup, таймаут 10 с, браузерный User-Agent
- **Извлечение контента:** `<title>`, meta description, текст из `<article>` / `<main>` / `<p>`, до 3000 символов
- **Минимум контента:** 100 символов, иначе ошибка
- **LLM:** таймаут 30 с, формат OpenAI Chat Completions, ключ из `.env`
- **Системный промпт:** загружается из `sprompt.md` (плейсхолдеры `{стиль}`, `{описание_стиля}`, `{текст_с_сайта}`)
- **Описания стилей:** подставляются из `Style.md`
- **Прокси:** если заданы `HTTP_PROXY` / `HTTPS_PROXY` в `.env`, используются при парсинге страниц и вызове LLM API
- **CORS:** включён для кросс-доменных запросов

## Обработка ошибок

Все ошибки возвращаются в поле `error` и отображаются в модальном окне:

| Ситуация              | Сообщение |
|-----------------------|-----------|
| HTTP 403              | Доступ к странице запрещён |
| HTTP 404              | Страница не найдена |
| HTTP 4xx (прочие)     | Сервер отклонил запрос |
| HTTP 5xx              | Проблема на стороне сайта |
| Таймаут загрузки (>10 с) | Превышено время ожидания ответа от сайта |
| Ошибка соединения     | Не удалось подключиться к сайту |
| Мало контента (<100)  | Недостаточно текстового контента |
| Таймаут LLM (>30 с)   | Сервис генерации не отвечает |
| Ошибка LLM API        | Текст ошибки от API |
| Пустой ответ LLM      | Сервис вернул пустой ответ |
